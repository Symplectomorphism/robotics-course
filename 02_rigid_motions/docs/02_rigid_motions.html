Coordinate Frames

Coordinate frames

  • A point is described by a coordinate vector.
    • The vector represents the displacement of a point w.r.t. some reference frame – we call this a bound vector.
    • Notice that a point is not the same as a vector!

  • A coordinate frame, is a triad of orthogonal axes which intersect at a point, known as the origin.

  • A real object comprises infinitely many points.

    • An object, unlike a point, has an orientation.
    • Attaching a coordinate frame to an object, every point within it can be described by a constant vector w.r.t. that frame.

  • This allows us to describe the position and orientation, the pose, of that object’s fixed frame w.r.t. a reference coordinate frame.

Coordinate frames


  • We think of AξB{^A}\xi_B as describing some motion: pick up {A}\{A\}
    • Apply a displacement and a rotation,
    • Now {A}\{A\} is transformed to {B}\{B\}.
  • If the initial superscript is missing, we assume that the pose is w.r.t. the world coordinate frame.
  • The point 𝐏\boldsymbol{P} can be described w.r.t. either coordinate frame: A𝐩=AξB•B𝐩. {^A}\bm{p} = {^A}\xi_B \bullet {^B}\bm{p}.
    • The operator •\bullet transforms the vector.
    • The new vector describes the same point but w.r.t. a different coordinate frame.

Relative pose

The point 𝐏\boldsymbol{P} can be descrbied by coordinate vectors relative to either frame {A}\{A\} or {B}\{B\}. The pose of {B}\{B\} relative to {A}\{A\} is AξB{^A}\xi_B.

Coordinate frames

 

  • Relative poses can be composed or compounded. AξC=AξB⊕BξC. {^A}\xi_C = {^A}\xi_B \oplus {^B}\xi_C.

Sub/super-script cancellation

If the subscript and superscript on each side of the ⊕\oplus operator match, then we can cancel out the intermediate sub/super-scripts. XξZ=XξY⊕YξZ. {^X}\xi_Z = {^X}\xi_Y \oplus {^Y}\xi_Z.

  • For this case, the point 𝐏\bm{P} can be described by A𝐩=(AξB⊕BξC)•C𝐩. {^A}\bm{p} = \left({^A}\xi_B \oplus {^B}\xi_C\right) \bullet {^C}\bm{p}.

  • Later, we will convert these abstract notions of ξ\xi, •\bullet and ⊕\oplus into concrete mathematical objects and operators.

Relative pose

The point 𝐏\boldsymbol{P} can be descrbied by coordinate vectors relative to either frame {A}\{A\}, {B}\{B\} or {C}\{C\}. The frames are described by relative poses.

Coordinate frames: more complex example

  • The fixed camera observes the object and estimates the object’s pose FξB{^F}\xi_B.

  • The other camera is not fixed, it is attached to the robot at some constant relative pose and estimates the object’s pose CξB{^C}\xi_B.

  • An alternative representation of the spatial representation is a directed graph.

    • Each node represents a frame and each edge represents a relative pose.
    • Recall: we can compose relative poses, e.g.: ξF⊕FξB=ξR⊕RξC⊕CξB,ξF⊕FξR=ξR. \begin{aligned} \xi_F \oplus {^F}\xi_B &= \xi_R \oplus {^R}\xi_C \oplus {^C}\xi_B, \\ \xi_F \oplus {^F}\xi_R &= \xi_R. \end{aligned}
    • Each equation represents a loop in the graph with each side of the equation starting and ending at the same node.

Algebraic rules of relative poses

Group properties

There are just a few algebraic rules:

ξ⊕0=ξ,ξ⊖0=ξ,ξ⊖ξ=0,⊖ξ⊕ξ=0, \begin{aligned} \xi \oplus 0 = \xi, \quad &\xi \ominus 0 = \xi, \\ \xi \ominus \xi = 0, \quad &\ominus \xi \oplus \xi = 0, \end{aligned}

where 00 represents a zero relative pose. A pose has an inverse

⊖XξY=YξX, \ominus {^X}\xi_Y = {^Y}\xi_X,

which is represented graphically by an arrow from {Y}\{Y\} to {X}\{X\}. Relative poses can also be composed or compounded

XξY⊕YξZ=XξZ. {^X}\xi_Y \oplus {^Y}\xi_Z = {^X}\xi_Z.

A relative pose can transform a point expressed as a vector relative to one frame to a vector relative to another X𝐩=XξY•Y𝐩. {^X}\bm{p} = {^X}\xi_Y \bullet {^Y}\bm{p}.

Composition is NOT commutative

ξ1⊕ξ2≠ξ2⊕ξ1, \xi_1 \oplus \xi_2 \neq \xi_2 \oplus \xi_1,

with the exception being the case where ξ1⊕ξ2=0\xi_1 \oplus \xi_2 = 0.

Algebraic manipulation of poses

  • The second loop eqn. in our example says that the pose of the robot is the same as composing two relative poses:
    • from the world frame to the fixed camera,
    • from the fixed camera to the robot.

⊖ξF⊕ξF⊕FξR=⊖ξF⊕ξRFξR=⊖ξF⊕ξR, \begin{aligned} \ominus \xi_F \oplus \xi_F \oplus {^F}\xi_R &= \ominus \xi_F \oplus \xi_R \\ {^F}\xi_R &= \ominus \xi_F \oplus \xi_R, \end{aligned}

which is the pose of the robot relative to the fixed camera, shown as the dashed line in the directed graph.

Writing these expressions by inspection

To find the pose of node XX w.r.t. node YY:

  • find a path from {Y}\{Y\} to {X}\{X\} and write down the relative poses on the edges in a left-to-right order,
  • if you traverse the edge in the direction of its arrow, precede it with the ⊕\oplus operator, otherwise use ⊖\ominus.

Rigid Motion in Two Dimensions

Working in two dimensions (22D)

  • A point is represented by its 𝐱\bm{x}- and 𝐲\bm{y}-coordinates (x,y)(x,y) or as a bound vector 𝐭=x𝐱+y𝐲. \bm{t} = x\bm{x} + y\bm{y}.

  • We wish to describe the red coordinate frame {B}\{B\} with respect to the blue coordinate frame {A}\{A\}.

  • Clearly the origin of {B}\{B\} has been displaced by the vector 𝐭=(x,y)\bm{t} = (x, y) and then rotated counter-clockwise by an angle θ\theta.

  • A concrete representation of the pose is therefore the 33-vector AξB∼(x,y,θ){^A}\xi_B \sim (x, y, \theta).

  • Unfortunately, this representation is not convenient for compounding since (x1,y1,θ1)⊕(x2,y2,θ2) (x_1, y_1, \theta_1) \oplus (x_2, y_2, \theta_2) is a complex trigonometric function of both poses.

  • We will look for a different way to represent the pose.

Properties of 22-D rotation matrices

  • They are orthonormal: each of its columns is a unit vector and columns are orthogonal.

  • The columns are the unit vectors that define the axes of the rotated frame {B}\{B\} w.r.t. {A}\{A\}.

  • It belongs to the special orthogonal group of dimension 22, i.e., 𝐑∈𝐒𝐎(2)\bm{R} \in \bm{SO}(2). This means that the product of any two such matrices belongs to the same group as its inverse.

Rigid Motion in Three Dimensions

Working in three dimensions (33D)

 

  • A point 𝐏\bm{P} is represented by its 𝐱\bm{x}-, 𝐲\bm{y}-, and 𝐳\bm{z}-coordinates (x,y,z)(x,y,z) or as a bound vector 𝐭=x𝐱+y𝐲+z𝐳. \bm{t} = x\bm{x} + y\bm{y}+ z\bm{z}.

  • We wish to describe the red coordinate frame {B}\{B\} with respect to the blue coordinate frame {A}\{A\}.

  • Clearly the origin of {B}\{B\} has been displaced by the vector 𝐭=(x,y,z)\bm{t} = (x, y, z) and then rotated in some complex fashion.

  • Our approach is again to consider an arbitrary point 𝐏\bm{P} w.r.t. each of the coordinate frames and to determine the relationship between A𝐩{^A}\bm{p} and B𝐩{^B}\bm{p}.

  • Rotation is much more complex for the 33D case so we’ll spend a good amount of time on it.

Properties of 33-D rotation matrices

  • They are orthonormal: each of its columns is a unit vector and columns are orthogonal.

  • The columns are the unit vectors that define the axes of the rotated frame {B}\{B\} w.r.t. {A}\{A\}.

  • It belongs to the special orthogonal group of dimension 33, i.e., 𝐑∈𝐒𝐎(3)\bm{R} \in \bm{SO}(3). This means that the product of any two such matrices belongs to the same group as its inverse.
